Independent Set in a tree

Vidhi Pareek Nov 24,2021




  • A set of nodes of a tree is such that no two of them are connected through an edge is called an independent set.
  • Given problem asks us to find out the size of maximum independent set i.e. finding the size of largest possible independent set.

  • Approach

    Two nodes will be connected only when one of them is a parent and the other is a child. In order to select nodes that are not connected through an edge we should remove all (parent,child) node pairs. This means for every node we have 2 choices:

    1. Select that node , ignore its children and process its grandchildren.
    2. Remove that node and process its children.
    3. We do the above two steps recurively and return the greater of the two values.
  • Formally , if `I(u)` represents the size of maximum independent subtree hanging from a node `u`
    `I(u)=max{1+sum I(w) , sum I(v)}`
    `w` refers to grandchildren of `u` while `v` refers to children of `u`

    Logic of the above formula : `1+sum I(w)` adds the current node and processes the grandchildren of `u` , `sum I(v)` removes the current node and processes the children of `u`. After processing, we take the maximum of both the values.

  • The following approach stores the output in `i` in struct node so that time complexity is reduced (By reducing computations of same problems) 
    //Only for binary tree
int I(node *root)
{
    if (root == NULL)
        return 0;

    if (root->i)
        return root->i;

    if (root->left == NULL && root->right == NULL)
        return (root->i = 1);

    // Calculate size excluding the current node
    int liss_excl = I(root->left) + I(root->right);

    // Calculate size including the current node
    int liss_incl = 1;
    if (root->left)
        liss_incl += I(root->left->left) + I(root->left->right);

    if (root->right)
        liss_incl += I(root->right->left) + I(root->right->right);

    // Maximum of two sizes is LISS, store it for future uses.
    root->i = max(liss_incl, liss_excl);

    return root->i;
}
  • Time Complexity for above approach : `O(n^2)`
