Activity Selection

Vidhi Pareek Nov 24,2021

  • Problem

    Given a set of activities, along with the starting and finishing time of each activity, find the maximum number of activities performed by a single person assuming that a person can only work on a single activity at a time.

  • Note:
    • We will use greedy approach because we want to maximize the count of activities that can be executed. We should greedily choose an activity with earliest finish time at every step, thus yielding an optimal solution.
    • The activities hat we pick should be non-conflicting activities.Two activities `i` and `j` are said to be non-conflicting if `s_i>=f_j` or `s_j>=f_i` where `s_i` and `s_j` are starting times while `f_i` and `f_j` are finishing times.

  • How to implement?

    1. Sort the given activities according to their finishing times.
    2. Select the first activity from sorted activities and add that activity to solution set.
    3. After the first activity, for further selections , move further in the list and check if the starting time of the selected activity is greater than or equal to the activity last added in solution set. If yes, add this activity to solution set otherwise skip it and move on to next activity.
    4. Keep repeating the previous step until you reach the end of the list.
      • Example: 

  • 
Activity-Selection(Activity, start, finish)
     Sort Activity by finish times stored in finish
     Selected = {Activity[1]}
     n = Activity.length
     j = 1
     for i = 2 to n:

             Selected = Selected U {Activity[i]}
             j = i
return Selected

  • Time Complexity Analysis

    Following are the scenarios for computing the time complexity of Activity Selection Algorithm

    Case 1: When a given set of activities are already sorted according to their finishing time, then there is no sorting mechanism involved, in such a case the complexity of the algorithm will be `O(n)`

    Case 2: When a given set of activities is unsorted, sorting algorithm will take O(nlogn) thereby increasing overall complexity to `O(nlogn)`
