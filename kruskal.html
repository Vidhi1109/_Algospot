Kruskal Algorithm

Vidhi Pareek Nov 24,2021

  • Problem

    Given a weighted, undirected graph. Find the minimum spanning tree of this graph.

  • What is a minimum spanning tree

    • A spanning tree of a graph is a subtree of this graph such that it contains all the vertcies of the main graph and all of them are connected.
    • Weight of a spanning tree is the summation of edge weights that make up that spanning tree.
    • Minimum spanning tree is a spanning tree whose weight is minimum `i.e.` summation of weight of its edges is minimum.

  • Approach

    Repeatedly add the next lighest edge that doesn't produce a cycle.
    • Sort all the edges in increasing order of weights.
    • Add edges to MST starting from least weighted. Ensure that adding an edge doesn't form a loop.

  • The Cut Property

    • Let `G=(V,E)` be a graph, and let `H \subset G` be a subgraph of `G` which is contained in a minimal spanning tree. Suppose `H` cuts `G` into a disjoint union `S ∪ T`.Then if `e \in E` is an edge from `S` to `T` `(in {G})` of minimal weight, then `H ∪ e` is contained in a minimal spanning tree. Note: `e \notin H`.
    • Proof: Assume that `H` is contained in minimal spanning tree `T`. Then `T` contains all the vertices of `G`, and it is connected, so there is an edge `e' \in T` from `S` to `T`. This edge does not belong to `H` by hypothesis
    • Claim: `T - {e'}∪{e}` is still a minimum spanning tree, which clearly contains `H ∪ {e}`.
      Now ,note that `T∪{e}` necessarily has a cycle.
      This cycle has an edge `e` going from `S` to `T`, so it must contain another edge `e'` going from `T` to `S`.
    • Now `T'=T - {e'}∪{e}` is connected (since removing on edge in a cycle does not change connectedness), and it is thus a tree because
      `|T'| = |T - {e'}∪{e}| = |T| = |V|-1.`
    • The claim is that it is a minimal spanning tree. But the weight of `T'` can be at most the weight of `T`, since `e'` has at least the same weight as `e`. It follows that `T` is a minimum spanning tree as well.
  • Disjoint Sets : Disjoint Sets is a data structure which partitions a set of items. A partition is a set of sets such that each item is in one and only one set. It has operations:
    1. makeset(x) - makes a set from a single item
    2. find(x) - finds the set that x belongs to
    3. union(x, y) - makes the union of the sets containing x and y
  • Implementation using trees :
    • The operation makeset is just to make a single node tree. The cost is Θ(1).
    • The operation union links the root of one tree to the root of the other tree. The cost is Θ(1).
    • The operation find requires traversing up the tree and costs Θ(h), where h is the height of the tree. The height could be on the order of the set size. To control the cost, the union should make the smaller tree in the union operation the sub tree of the larger tree. This is union by size (by set size) or union by rank (by tree height). Naturally this requires storing the tree size or height in the root. Using union by size or rank the height of tree is logarithmic with the number of unions (in other words the tree/set size).
    • The cost for n-1 unions and m finds is `O(n + m lg n)`
    • Much better can be done by path compression : Path compression makes every node encounter during a find linked with the root directly. Then a sequence of n-1 unions and m finds is only slightly more than linear in n and m. 
  • 
Algorithm Kruskal(G)
    sort E by the edge weights // Note this is a Priority Queue
    make disjoint sets of vertices, d(v)
    while ecounter < |V|-1 and E is not empty do
          (u, v) ← remove minimum from E
          if find(u) ≠ find(v) then
              T ← union(u, v)
              ecounter++
return T
